DeepSeek-V4-Flash 项目概览

探索一个支持百万token上下文的高效MoE大语言模型

什么是 DeepSeek-V4-Flash?

DeepSeek-V4-Flash 是 DeepSeek-AI 推出的新一代 MoE（混合专家）语言模型。它拥有 284B 总参数，但每次推理仅激活 13B 参数，在保持高性能的同时大幅降低了计算成本。

MoE架构

MoE 配置参数

n_routed_experts: 256 可路由专家总数，每个token动态选择
n_shared_experts: 1 共享专家，所有token都会经过

score_func: sqrtsoftplus 路由评分函数，决定token分配

生成函数核心代码

Python def generate( model, prompt_tokens, max_new_tokens, eos_id ): # Prefill阶段 logits = model.forward( tokens[:, prev_pos:cur_pos], prev_pos ) # Decode阶段循环 for cur_pos in range(total_len): next_token = sample(logits) tokens[:, cur_pos] = next_token if next_token == eos_id: break return tokens
自然语言解释
定义generate函数，接收模型、prompt tokens等参数
Prefill阶段：一次性处理所有输入token，填充KV缓存
Decode阶段：循环生成新token
对logits采样得到下一个token
检查是否达到结束符(eos_id)
返回完整的token序列

采样策略

🎲

温度采样

temperature=0 → 确定性输出，选择概率最高的token

🎯

Top-K采样

只从概率最高的K个token中选择，平衡多样性和质量

📊

Top-P采样

累积概率达到P时停止，动态调整候选token数量

⚡

Gumbel-Max

高效采样方法，避免GPU到CPU同步，提升生成速度

分布式词汇表

DeepSeek-V4-Flash使用分布式词汇表，将词汇表分片存储在不同的GPU上，支持超大规模模型的训练和推理。

编码系统代码解析

Python class ParallelEmbedding(nn.Module): def __init__(self, vocab_size, dim): self.vocab_size = vocab_size self.dim = dim # 计算分片大小 self.shard_size = vocab_size // world_size self.embedding = nn.Embedding( self.shard_size, dim ) def forward(self, tokens): # 收集所有GPU上的嵌入 return all_gather(self.embedding(tokens))
自然语言解释
定义ParallelEmbedding类，用于分布式词嵌入
计算每个GPU负责的词汇表分片大小
每个GPU只存储自己负责的那部分嵌入矩阵
forward时通过all_gather收集所有GPU的结果

词汇表特殊Token

[PAD] = 0 填充token，用于对齐不同长度的序列
[EOS] = 1 结束符token，表示文本生成结束
[SOS] = 2 开始符token，标记序列开始
[MASK] = 3 掩码token，用于预训练任务

量化策略

DeepSeek-V4-Flash采用FP4/FP8混合量化策略，在保持模型质量的同时大幅降低显存占用。

FP16（传统）
精度 高
显存占用 高
速度 较慢
FP4/FP8（Flash）
精度 接近FP16
显存占用 降低75%
速度 显著提升
